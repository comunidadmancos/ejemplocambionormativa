PROPUESTA

NORMATIVA DE REGISTRO DE MULTICUENTAS.



1. CONSIDERACIONES GENERALES

El registro de multipersonajes se realiza para poder llevar un control de los usuarios que utilizan varias cuentas de Steam para rolear diferentes personajes (PJs) dentro del servidor.

1.0. Para poder optar a esta posibilidad incluso en el servidor abierto, será necesario disponer de whitelist aprobada o haber dispuesto anteriormente del rango de creador de roles y no haberlo perdido por sanción.

1.1. Existirán las siguientes incompatibilidades entre diferentes PJs de un mismo usuario:
     •    Fuerzas policiales (LSPD/LSSD) y organización criminal (empleado/miembro).
     •    Fuerzas policiales (LSPD/LSSD) y banquero/armero/narcotraficante.
     •    Banquero/armero/narcotraficante y organización criminal (empleado/miembro).
     •    Pertenecer a 2 organizaciones criminales diferentes.
     •    Pertenecer a 2 empresas legales diferentes si comparten función o actividad.

1.2. Los PJs no se podrán conocer ni tener relación entre sí directa o indirectamente, ni conseguir ningún tipo de privilegio ni beneficio material derivado del rol o las relaciones de otro de los PJs propios. 

1.3. Se deberá de tener especial cuidado con diferenciar los PJs y evitar el MG en caso de compartir relaciones personales IC entre diferentes PJs propios. En caso de duda razonable podrán aplicarse limitaciones a criterio del staff.

1.4. Cada PJ deberá de estar vinculado a su propia cuenta de Steam, puesto que si se realiza un CK a alguno de ellos, se borrará todo lo que le pertenezca y esto no podrá ser reclamado.

1.5. Queda prohibido traspasar o intentar traspasar pertenencias entre PJs propios, aunque sea a través de intermediarios. Se considerará especialmente grave si se hace para evadir pérdidas materiales en caso de CK.

1.6. Cada usuario podrá tener un máximo de tres (3) PJs diferentes, salvo autorización administrativa expresa.

1.7. Para poder usar esta posibilidad será necesario rellenar la plantilla con los datos que se piden. En caso de usar varias cuentas sin haber publicado los datos que se piden, primero se dará un aviso, y en caso de persistir se expulsará temporalmente a una o todas las cuentas.

1.8. Cada usuario deberá tener un post individual en la sección apropiada del foro, con cada una de las cuentas y los datos solicitados en la plantilla (cuentas de Steam, nombres IC y una PEQUEÑA historia o descripción del rol planeado para cada PJ). 

1.9. Cuando un PJ reciba un CK, deje de usarse o reciba un cambio importante a sus datos (cambio de nombre, cambio de rol principal, etc), el usuario deberá actualizar la plantilla del PJ en cuestión reflejando de la forma más resumida posible estos cambios. Bajo ningún concepto se puede eliminar la plantilla de un PJ que ya haya adquirido cualquier tipo de relación personal o notoriedad IC.

1.10. El staff podría llamar al usuario para interrogarle en caso de sospechas de mal uso o comisión de infracciones.

1.11. El uso de multicuentas con la intención de evadir rol o evadir sanciones, se considerará un agravante de gravedad media o alta según las circunstancias.
